# FastAPI & Backend Engineering Study Plan

## From Zero to Confident Junior Backend Engineer in 8–10 Weeks

This curriculum takes you from knowing nothing about servers to building, testing, and deploying a production-ready FastAPI backend. Every module builds on the previous one. Do not skip ahead.

---

## Prerequisites

- A computer running macOS, Linux, or Windows (with WSL2)
- Basic Python knowledge (variables, functions, loops, classes)
- A text editor (VS Code recommended)
- Willingness to use the terminal exclusively — no GUI shortcuts

If you lack basic Python, spend 1 week on [Python basics](https://docs.python.org/3/tutorial/) before starting.

## How to Use This Curriculum

1. **Read each module top to bottom.** Do not skim.
2. **Type every command yourself.** Do not copy-paste terminal commands.
3. **Complete every exercise.** The exercises are the learning — the text is context.
4. **Answer every checkpoint quiz** before moving on. If you get one wrong, re-read.
5. **Build the mini-projects.** They cement understanding.
6. **Take notes in your own words.** Rewriting forces comprehension.

- Explain how a web request travels from browser to server and back
- Design RESTful APIs following industry conventions
- Navigate Linux systems confidently from the terminal
- Use Git for real-world collaborative development
- Build FastAPI applications with proper validation, auth, and database integration
- Write tests for your APIs
- Containerize and deploy your applications
- Debug production issues systematically

---

## Teaching Philosophy

This curriculum follows three principles:

1. **WHY before HOW.** You will understand the reason a thing exists before you learn to use it.
2. **First principles over recipes.** You will build mental models, not memorize commands.
3. **Intensity with structure.** This is not casual reading. It is a focused, hands-on program.
